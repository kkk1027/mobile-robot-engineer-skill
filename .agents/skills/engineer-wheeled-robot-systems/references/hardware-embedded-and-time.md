# 硬件、嵌入式与时间

## 计算职责

| 平台 | 适合职责 | 避免 |
|---|---|---|
| MCU | 电流/速度环、编码器采样、PWM、硬件 watchdog、急停输入 | 大型 ROS 图、非确定性任务 |
| Jetson | 相机、深度学习、GPU 加速感知、局部规划 | 让 GPU 负载阻塞底盘停机链 |
| 工控机 | ROS 主系统、多网口、现场总线、长期运行 | 未验证驱动和内核时声称硬实时 |

把安全停止留在离执行器最近且不依赖 DDS/网络的层级。

## 接口选择

- UART/USB serial：简单但要处理设备名、波特率、校验、重连、粘包和权限。
- CAN/CAN-FD：适合分布式驱动；定义 arbitration ID、字节序、缩放、心跳和 bus-off 恢复。
- Ethernet：适合高带宽传感器；处理 MTU、交换机、组播、防火墙和 PTP。
- EtherCAT：适合确定性现场总线；验证主站周期、分布式时钟、PDO 和故障状态。
- micro-ROS：适合受限 MCU；明确 agent 传输、实体生命周期、内存策略和断链行为。

不要仅因为接口能通信就认为满足控制实时性。

## 控制与采样预算

从任务验收反推频率：

- 电流环通常在驱动器或 MCU 的最高频层。
- 轮速闭环通常在 MCU/驱动器，频率显著高于上位机导航控制。
- 上位机底盘命令、状态估计、局部规划和感知各自定义频率与延迟预算。

用以下关系检查：

~~~text
端到端年龄 = 采样等待 + 传输 + 排队 + 计算 + 控制周期等待
~~~

同时记录平均值、P95/P99、最大值和丢包，不只报告平均频率。

## 时间同步

- 单机使用同一单调时钟衡量延迟。
- 多机优先 PTP；NTP 仅在精度满足测量要求时使用。
- MCU 时间戳在采样时生成，维护与 ROS 时间的映射和漂移估计。
- 仿真统一 use_sim_time，确保所有节点订阅 /clock。
- 回放 rosbag 时区分记录时间、源时间戳和处理时间。

发现负延迟、未来 TF 或周期跳变时先检查时间源。

## 硬件契约

对每个设备记录：

- 型号、固件、接口、供电、地址和唯一标识。
- 输入输出范围、单位、符号、饱和和分辨率。
- 标称/最低频率、最大允许陈旧时间和重连策略。
- 启动、自检、正常、降级、故障和恢复状态。
- watchdog、心跳、错误码和安全输出。

## 实时与线程

- 将控制回调、硬件读写和非实时日志分离。
- 避免在实时路径分配内存、阻塞 IO、DNS、磁盘日志或加载模型。
- 明确 executor、callback group、线程优先级和 CPU affinity。
- 测量调度延迟后再决定 PREEMPT_RT、锁内存或实时优先级。
- 防止优先级反转，并为日志/诊断设置限流。

## 上电与停机

定义：

1. 硬件安全默认状态。
2. 驱动连接和固件兼容检查。
3. 传感器和时间同步就绪。
4. 状态估计收敛。
5. 控制器激活。
6. 自动任务获准。

停机反向执行，先撤销运动授权并确认速度为零，再释放软件资源。通信中断、电池过低、编码器异常或控制器崩溃时必须进入预定义安全状态。
