# 轮式底盘运动学

## 通用检查

先定义 REP-103 坐标：x 向前、y 向左、z 向上，角速度绕 z 轴逆时针为正。确认轮径、轮距、轴距的测量基准和单位均为 SI。

所有底盘都必须明确：

- 控制自由度和不可积约束。
- 轮子驱动、转向和被动滚动关系。
- 正逆解、速度/加速度/转角/转角速度限制。
- 里程计来源、打滑模型和标定参数。
- 指令超时后的安全行为。

## 差速驱动

适用于左右两个独立速度组、不可侧移的底盘。理想模型：

~~~text
v = r(ωr + ωl)/2
w = r(ωr - ωl)/L
ωr = (v + wL/2)/r
ωl = (v - wL/2)/r
~~~

r 为有效轮半径，L 为有效轮距。四轮差速通常把同侧轮视为一组，但必须验证机械一致性和负载分配。

优先映射 diff_drive_controller。重点检查轮名、轮径倍率、轮距倍率、open_loop、position/velocity feedback 和 cmd_vel 超时。

## 滑移转向

四轮或履带等效滑移底盘可使用差速命令接口，但理想差速模型会低估转弯滑移。不要把几何轮距直接当作有效轮距。通过低速直线、定角速度旋转和组合轨迹标定有效参数，并在状态估计中反映更大的横向/航向协方差。

高附着地面原地转向可能带来大电流、轮胎磨损和控制非线性；为角速度和加速度设置更严格限制。

## 阿克曼和自行车模型

车轮转向且不能原地转向时使用。自行车模型近似：

~~~text
w = v tan(δ) / L
R = L / tan(δ)
~~~

δ 为等效转向角，L 为轴距。真实左右转向轮应满足不同内外轮角。低速路径跟踪与高速车辆动力学不是同一问题；高速场景要加入侧偏、轮胎、制动距离和延迟模型。

优先使用 ackermann_msgs 表达速度和转角目标，并验证控制器接收的是方向盘角、等效前轮角还是执行器位置。

分别标定速度→电机转速和转向角→舵机位置的增益、偏置、饱和与死区。优先使用实测转向角计算里程计；若只能使用转向命令近似，必须考虑舵机延迟、饱和和间隙，提高航向协方差并用外部轨迹验证。

## 麦克纳姆

支持平面 vx、vy、wz。矩形四轮且滚子方向正确时可用标准逆解，但符号取决于轮子安装和滚子朝向。

上线前逐轮悬空测试，建立“正向轮速—底盘速度”矩阵；不要仅依据 CAD 命名推断符号。里程计对轮径误差、载荷和地面摩擦敏感，应与 IMU/视觉/雷达融合。

优先使用 mecanum_drive_controller 或经验证的等价控制器，明确轮序和接口类型。

## 全向轮

三个或更多全向轮通过雅可比矩阵完成底盘速度与轮速映射。根据每个轮接触点位置和驱动力方向构造矩阵，检查矩阵秩和条件数。接近奇异或几何布置不良时限制命令空间。

不要把三轮全向和四轮麦克纳姆参数互换。

## 舵轮与独立转向模块

每个模块同时具有驱动角速度和转向角。求解时：

1. 将底盘 twist 转换为各模块接触点速度。
2. 选择转向角与轮速符号组合，最小化转角变化。
3. 处理零速时的角度保持、转角限位和跨越 ±π。
4. 在高层命令变化时限制模块转角速度和轮加速度。

需要专用 swerve/steering controller 或项目控制器；不得退化成普通麦克纳姆配置。

## 特殊轮式底盘

- 摇臂悬架：底盘平面运动可近似差速，但姿态、接地和轮速分配需额外建模。
- 铰接转向：以铰接角而非前轮转角建模，检查车体两段 TF。
- 多轴转向：明确同相、反相和蟹行模式及模式切换状态机。
- 拖挂系统：规划和控制需考虑非完整约束、折叠角和倒车不稳定性。

## 选择与验证

至少验证：

- 单轮正方向和编码器符号。
- 直线距离、原地或最小半径转弯、方形/八字轨迹。
- 低速死区、静摩擦、饱和、制动和反向间隙。
- 指令与反馈时间戳、控制周期抖动和超时停机。
- 里程计相对外部基准的尺度和航向误差。
