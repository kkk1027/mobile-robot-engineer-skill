# 轮式机器人问题路由

先根据现象选择一个或多个问题族，再读取相应领域参考文件。下表按风险排序；“高频”表示 ROS 生态中反复出现的工程类别，不代表未经测量的全球百分比。

| 问题族 | 典型现象 | 最低风险检查 | 常见伪修复 |
|---|---|---|---|
| P0 命令与安全 | 多写者、残余零速、急停无效、切换抖动 | 发布者/订阅者、仲裁状态、超时归零、停止屏障 | 再加一个 `cmd_vel` 发布者或无限重发零速 |
| P0/P1 运动学/反馈 | 反向、跑偏、无 odom、横移错误 | 关节方向、轮径/轮距、单位、命令—轮反馈闭环 | 只反转显示坐标或调大 PID |
| P0/P1 TF/时间 | 断链、双写、旋转、future extrapolation | owner、frame、时间源、TF age、外参 | 重复发布静态/动态 TF 掩盖错误 |
| P1 定位融合 | 漂移、跳变、启动不定位 | 输入轴向、协方差、时间戳、map/odom 语义 | 盲目放大 covariance 或 timeout |
| P1 导航/action | active 但不完成、取消异常、恢复循环 | action trace、lifecycle、goal/取消来源、错误码 | 只检查节点 active 或无限 retries |
| P0/P1 障碍接管 | 感知失效、切换空窗、碰撞/卡死 | 传感器→costmap→决策→仲裁、注入障碍 | 多控制源直接争用底盘 |
| P1 QoS/网络 | Topic 同名却收不到、跨机发现失败 | 类型、namespace、QoS offered/requested、RMW/domain/接口 | 只改 topic 名或无限增大队列 |
| P1 驱动/设备 | USB/串口变化、断线退出、陈旧反馈 | 稳定设备 alias、连接状态、重连、fake/回放 | 依赖 `/dev/ttyUSB*` 顺序或无限 busy-loop |
| P1/P2 性能 | 频率低、RTF 低、CPU 未满却卡顿 | 墙钟频率、延迟、RTF、队列、上下文切换、GPU | 把配置 rate 当实测或默认上实时优先级 |
| P1/P2 仿真 | URDF 可见但不动、传感器假数据、sim/real 时钟乱 | 插件输入、关节/碰撞/惯量、`/clock`、运行图 | 用高摩擦/无噪声掩盖控制缺陷 |
| P2 配置/版本 | 参数未加载、namespace 冲突、依赖漂移 | 参数 dump、launch 参数、提交、依赖版本 | 复制另一发行版 YAML |
| P2 回归/证据 | 人工操作才通过、异常无法复现 | launch test、rosbag、工件、运行 observation | 只保留最终截图或成功日志 |

## 通用诊断输出

每个问题族输出以下六项：现象、假设、预计观测、最低风险检查、结论、回归测试。P0 检查先验证停止路径；任何实机变更仍服从 G2–G4 授权。

## 任务模式切换

导航、遥控、跟随、探索、回充、避障或恢复之间切换时，要求定义：旧所有者、停止屏障、接管条件、新所有者、输入 freshness、失败出口和恢复条件。没有这些内容时，标为 `blocked`/`partial`，而不是假设 ROS 发布时序会正确解决竞争。
