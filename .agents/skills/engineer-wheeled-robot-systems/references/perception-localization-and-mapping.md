# 感知、定位与建图

## 从任务选择传感器

- 二维室内导航：2D LiDAR + 编码器 + IMU 常为基础组合。
- 室外：加入 GNSS/RTK，必要时使用 3D LiDAR、相机或轮速约束。
- 低矮/悬空障碍：使用深度相机或 3D LiDAR，不依赖单平面雷达。
- 视觉语义：确认光照、运动模糊、算力和隐私约束。

为每个传感器记录 frame、时间戳、频率、量程、协方差、标定和故障语义。

## 标定顺序

1. 验证驱动方向、编码器尺度和 IMU 坐标。
2. 标定相机内参。
3. 标定传感器相对 base_link 的外参。
4. 验证多传感器时间同步。
5. 用独立轨迹检查组合误差。

不要通过调整 EKF 协方差掩盖错误外参或时间戳。

## 状态估计

区分连续局部状态与全局修正：

- odom 系：连续、短期平滑，可漂移。
- map 系：全局一致，允许因定位修正发生变化。

robot_localization 或自定义融合器配置时检查：

- 同一信息是否重复融合。
- 差分/相对模式是否符合消息含义。
- 二维模式是否错误忽略坡度或 IMU 安装。
- 输入协方差是否真实且非零占位。
- GNSS 使用时坐标转换、磁偏角和天线杆臂。
- 测量延迟和 out-of-sequence 数据处理。

## SLAM 与定位

选择算法前确认环境、传感器和地图表达：

- 2D LiDAR 室内：SLAM Toolbox、Cartographer 等。
- 视觉/视觉惯性：关注纹理、曝光、回环和尺度。
- 3D LiDAR：关注运动畸变、点云去畸变和算力。
- 已知地图：AMCL 或相应定位器，并定义初始位姿流程。

验收不只看地图图像；测量闭环误差、重定位成功率、轨迹误差、CPU/内存和地图可导航性。

## 障碍与代价地图

- 观测源 frame 和时间戳必须可变换。
- obstacle、voxel、inflation 等层与机器人 footprint 一致。
- clearing 与 marking 范围符合传感器视场。
- footprint 使用真实碰撞包络，并考虑载荷变化。
- 动态障碍需要合理 persistence 和衰减。

机器人“看见障碍但不避让”时分别检查原始传感、TF、observation source、costmap 层、规划器和控制器，不要直接增大 inflation。

## 感知性能

建立预算：

~~~text
测量时刻 → 驱动发布 → 变换可用 → 感知输出 → 代价地图 → 控制命令
~~~

记录端到端年龄。GPU 推理帧率高但结果陈旧时仍不可用于高速控制。对队列堆积使用丢旧保新策略，并在输出中携带源时间戳。

按任务模式关闭高负载 RGB、骨架、点云或推理时，必须同时定义设备健康探测、队列清空、重连退避、恢复首帧和失败回滚。推理引擎可以常驻但输入队列应有界；恢复后不能先消费停用期间积压的旧帧。身份检测负责目标准入时，还要定义检测框与深度/骨架目标的关联、遮挡保持、重捕获和“不得自动改跟陌生目标”的验收。

多个无名称 Range 通道在用于安全前必须逐通道建立物理方向、安装位姿、量程、刷新率和掉线语义；生成映射记录并回填 TF。只报告信标方向或码值的接收器不能按距离传感器使用。

## 故障降级

定义雷达、相机、IMU、GNSS、编码器分别掉线时：

- 能否继续任务。
- 允许的速度和持续时间。
- 是否切换定位源。
- 何时停止并请求人工。

降级必须可观测，不能静默降低安全性。
